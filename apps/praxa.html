praxa

A terminal AI coding agent that runs on your own machine — built to do what Claude Code does, with no API key and no data leaving your box.
~/anything $ praxa
▸ indexing repo via PEP spreading-activation…
▸ context selected: 7 files, 2 tests
praxa › add a rate limiter to the API
▸ editing api/limits.py … running tests … ✓ 12 passed

What makes it different

Its moat is PEP spreading-activation context selection — instead of dumping your whole repo at a model, Praxa radiates activation from your request through a weighted graph of your code and pulls in only what matters. Cheaper, sharper, and it runs against a small local model.

How to launch it

It's a terminal tool, so it lives in your shell, not this page. From anywhere:

$ praxa

(Launcher installed at ~/.local/bin/praxa; engine in pep.agent. Verified on qwen2.5:7b.)
